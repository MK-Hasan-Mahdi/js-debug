Javascript Q&A

(1)How does Javascript works?

Ans: Javascript is a single-threaded, high level programming language. It considered high level because it’s not required direct interaction with computer operating system or hardware to run code. There are two phases in javascript code executions level. Variable environment and code of execution, in variable environment variable are assigned in memory without initializing. In execution environment executes code line by line synchronous way. When a asynchronous function invoked another, execution environment also works together. It rendering program like call stack. DOM parser, CSS parser works with Javascript engine like V8 engine, spider monkey etc. And all this happens together and gives us human readable form.

(2)What does do event loop?

Ans: Event loop is responsible for executing code, collecting and processing events and executing queued sub-tasks. Synchronous code executes first and simultaneously asynchronous codes also remaining to executions. For example – if we execute 3 functions, first and last functions are synchronous and second function is asynchronous. Synchronous code come from call stack and executes, but when an asynchronous code come it pushed to event table, event table is responsible for asynchronous code to the event queue after a specified time but in the meantime third code executes before asynchronous code executes. Event loop continuously checks call stack. From event table when come second asynchronous code and run it. Javascript though a single thread language but its execution do faster for browsers web-api and for node js c++.

(3)Difference between local storage and session storage?

Ans: HTTP protocol does not track data for client response and request by server and client, so to solve this problem local storage, session storage and cookies concepts come. Below are some difference between local storage and session storage.

    Local storage:
  • local storage data not expired until we remove or expires.
  • Local storage data kept in browser though we close the browser.
  • It is accessible from any window.
    Session storage:
  • Session storage data will be closed when we close window or specific tab.
  • Session storage remains until we close specific session, but not stored in local storage.
  • It is accessible from same tab.
